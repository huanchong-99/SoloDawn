## 1. 本次补齐项

### 1.1 Final Integration Repair 输入完整化

`Final Integration Repair` task/terminal 的输入现在补齐：

- final gate 名称：artifact readiness / branch / repo / terminal enforce blockers。
- `quality_run_id`：branch/repo quality gate 失败时写入 repair prompt 和 repair task 描述。
- blocker 原文：保留质量门输出的阻断项、相关文件和修复建议。
- 原始用户目标与已完成 task 摘要。
- 强制要求真实构建、真实测试、入口 smoke、跨任务集成修复，并提交修复 commit。

### 1.2 重复 blocker 防无限循环

新增稳定 fingerprint：

- 对 `gate_level + blocker reason` 做归一化。
- 写入 repair task 描述：`Final-Repair-Fingerprint: <hash>`。
- 后续同一 blocker 再次出现时，repair prompt 自动升级，要求先复现失败、分析上一轮为何无效、改变修复策略。
- 超过 `FINAL_REPAIR_MAX_ROUNDS` 仍未清除 blocker 时，workflow 进入 `failed` 并保留诊断。

### 1.3 Workflow detail reconciliation

工作区详情 API 现在会检查：

- workflow 仍为 `running`；
- 所有 task 已经是 completed / failed / cancelled；
- 所有 terminal 已经是 completed / failed / cancelled；
- 没有 starting / waiting / working / quality_pending terminal。

满足条件时，详情查询会触发 orchestrator runtime 的 completion reconciliation。若 agent 仍在线，直接运行 final gates；若 agent 不在线但可从持久化状态恢复，则先恢复 agent 再运行 reconciliation。

### 1.4 UI 展示语义

当 workflow 为 `running` 且存在未结束的 `Final Integration Repair` task 时：

- 工作流详情页状态显示为 `repairing final issues`。
- 工作区对话进度面板也显示 `repairing final issues`。
- 这只是展示语义，不新增数据库状态，避免破坏既有 workflow status 枚举和 API 兼容性。
