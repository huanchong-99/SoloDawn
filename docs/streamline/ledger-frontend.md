# Frontend Streamline Ledger

Synthesis of census findings across 31 frontend unit agents. Candidates deduplicated by path/symbol (strongest evidence + widest blast radius kept), cross-linked into feature clusters, and ranked by removal priority.

**Tooling note:** `fast-context` MCP (`mcp__fast-context__fast_context_search`) returned `resource_exhausted` errors on EVERY unit agent's calls (all 31 units). 100% fast-context unavailability for this area. All cross-file usage analysis was performed via Grep (ripgrep) fallback. Grep results are reliable for static import/symbol analysis; dynamic/string-keyed dispatch and barrel re-exports may have minor gaps.

Priority key: P0 = remove first (high confidence, isolated, delete). P1 = high-value cleanup. P2 = coordinated multi-file / investigate. P3 = low-impact / keep-for-now.
