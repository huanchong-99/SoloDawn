## Invisible Features

- **Codex review mode** (`spawn_review` + `codex/review.rs`): A full separate execution path for triggering AI code-review sessions via Codex. Not surfaced in obvious UI routes beyond `task_attempts.rs::spawn_review`. Active (called from orchestrator review flow).
- **Cursor MCP auto-trust bootstrap** (`cursor/mcp.rs`): Before each Cursor spawn, SoloDawn silently writes `mcp-approvals.json` so the agent can access configured MCP servers without interactive prompts. This is invisible to users.
- **Codex approval relay** (`codex/client.rs::handle_server_request`): When Codex's app-server requests approval to run a command or apply a patch, this client relays the decision from `ExecutorApprovalService` and queues denial feedback as a follow-up user message. This approval bridge is invisible in the executor surface.
- **Pending feedback queue** (`codex/client.rs::pending_feedback`): If approval is denied before the conversation_id is registered, denial feedback is buffered and flushed on `register_session`. This retry mechanism is invisible.
- **Codex session forking** (`codex/session.rs::fork_rollout_file`): For follow-up sessions, Codex rollout files are copied with a new UUID to avoid mutating the original session history. Invisible to users.
- **`CommandExecutionRequestApproval`/`FileChangeRequestApproval` dead branches** (`codex/client.rs:242-251`): These server-request variants are commented "unreachable until switching to v2 APIs". They log an error and return failure — dead code shim for a future protocol version.
- **`replace_all` field in `CursorStrReplace`/`CursorMultiEditItem`** (`cursor.rs:1208,1223`): Deserialized from Cursor's JSONL but never read anywhere in the processing logic. The field exists purely for serde completeness.
- **`CODEX_NPX_VERSION` pin** (`codex.rs:256`): Hard-pins `@openai/codex@0.77.0`. Any version bump requires a code change. Invisible configuration.
- **NPM stderr noise filter** (`codex/normalize_logs.rs:366`): `NPM_NOISE_RE` silently drops `npm warn`/`npm notice` lines from Codex stderr output before display.
