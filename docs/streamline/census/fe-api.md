# Census: unit `fe-api`

Scope: `frontend/src/lib/api.ts` (1639 lines) + `shared/types.ts` (818 lines, generated).
Branch: `refactor/streamline-quality-gates`.

`api.ts` is the single typed HTTP client for the whole frontend. Every API group is a
plain object literal of `async` methods that build a URL, call `makeRequest` (or raw
`fetch` for multipart), and pipe the `Response` through `handleApiResponse` /
`handleApiResponseAsResult`. `shared/types.ts` is the auto-generated TS mirror of the
Rust `ts-rs` types (do not hand-edit) and supplies almost every request/response type
`api.ts` imports.

## Flagged items

### BUG — uploadAuditDoc multipart field mismatch (`api.ts:605-617`)
Frontend appends `formData.append('audit_doc', file)`. Backend
`crates/server/src/routes/planning_drafts.rs:1071-1074,1135-1137` only accepts a field
named `file` and otherwise returns 400 "No 'file' field found in multipart upload".
Every audit-doc upload fails. Fix: change to `formData.append('file', file)`.
Confirmed against backend handler (grep fallback; fast-context quota-exhausted).

### G1 candidate — "open in external editor/IDE" surface
`projectsApi.openEditor` (329), `attemptsApi.openEditor` (791), `repoApi.openEditor`
(1113), `configApi.checkEditorAvailability` (1152), plus local `OpenEditorApiRequest`
(99-101). Live callers exist: `useOpenInEditor.ts`, `useOpenProjectInEditor.ts`,
`useEditorAvailability.ts`, `DiffCard.tsx:271`, `ui-new/actions/index.ts:650`,
`GitPanelContainer.tsx:214`. Real blast radius — not a free delete. Disposition:
investigate (only cut as a coordinated G1 change incl. those callers + EditorType usage).

## Invisible / non-obvious features
- **planningDraftsApi confirm→materialize flow + AuditPlan (System B)**: inline
  `PlanningDraftResponse` carries `auditPlan`/`auditMode`/`auditDocPath` and `feishuSync`.
  `confirm` posts `{retainBuiltin}`, `materialize` creates a workflow. Backend in
  `crates/server/src/routes/planning_drafts.rs`.
- **feishuApi**: full Feishu (Lark) integration client (status/config/reconnect/test).
- **scratchApi.getStreamUrl / makeRequest WS**: returns a `…/stream/ws` URL string only.
- **Quality Gate System A types** live in `shared/types.ts` (`QualityRun`,
  `QualityIssueRecord`, `QualityRunDetail`, `WsEventType "quality.gate_result"`,
  `ExecutionProcessRunReason "qualityscan"`) but `api.ts` exposes **no** quality-gate
  HTTP group — quality data reaches FE via WS events / embedded in other responses.
