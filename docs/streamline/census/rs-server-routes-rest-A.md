## Key Findings

### Candidates for keep/cut review

1. **`oauth.rs` handoff/token stubs** — `handoff_init`, `handoff_complete`, `logout`, `get_token`, `get_current_user` are all hard `BadRequest` stubs. Frontend still calls `/auth/handoff/init`, `/auth/token`, `/auth/user`, `/auth/status`. The `status` handler is real. Others are dead code in the handler body but the routes are registered. Kind: `stub`.

2. **`cli_types.rs` placeholder types** — `CliInstallHistory` and `CliDetectionCache` are local placeholder structs (lines 43–67) with a commented-out `// use db::models::cli_install_history::{CliInstallHistory, CliDetectionCache}`. The real DB types exist at `crates/db/src/models/cli_install_history.rs`. Kind: `redundant`. The local placeholders should be replaced by the db imports.

3. **`cli_types.rs` unimplemented endpoints** — `get_install_status` (line 499), `get_install_history` (line 530), `get_cached_status` (line 546) all return `ApiError::NotImplemented`. Kind: `stub`. No frontend callers found for `get_cached_status` or `get_install_history`.

4. **`event_bridge.rs` `SharedEventBridge` type alias** — `pub type SharedEventBridge = Arc<EventBridge>` (line 85) is exported but not used anywhere outside the file. Kind: `dead`. The `EventBridge` itself is used; only the type alias is unused.

5. **`config.rs` `REMOTE_FEATURES_ENABLED = false`** — Hard-coded `false` constant that disables cloud org/sharing features. Frontend gates 15+ features on this. Kind: `legacy` / disabled feature flag. Not dead — it is actively checked.

6. **`filesystem.rs` `pick_folder` on non-Windows** — The non-Windows stub (line 144) returns `ApiError::Internal("Folder picker is only supported on Windows")`. No frontend TS caller found for `/filesystem/pick-folder` at all. Kind: `dubious-feature`.

7. **`oauth.rs` `#[allow(dead_code)]`** — `HandoffInitPayload` and `HandoffCompleteQuery` are annotated `#[allow(dead_code)]` (lines 52, 74) because they're only used in stub handlers. Reinforces stub status.
