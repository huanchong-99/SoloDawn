## Invisible features

- **Redis message-bus backend** (`message_bus.rs`): full `RedisBus` PubSub impl for multi-container deployments, selected via `SOLODAWN_MESSAGE_BUS=redis` + `SOLODAWN_REDIS_URL`. The selector `MessageBus::from_env` is never called — production uses `MessageBus::new(1000)`. Capability exists but is unreachable through current wiring.
- **Crash recovery + resume cursor** (`runtime.rs`): `recover_running_workflows` + `resume_cursor_for_workflow` rebuild orchestrator agents from persisted state on startup and seed the GitWatcher from the newest commit across `git_event` ∪ `quality_run` (R8-C2) so handoff commits aren't skipped. R7-PB1: never auto-fail on restart — drop to `paused`.
- **System B scoring audit** (`types.rs`): `AuditPlan`/`AuditMode`(Builtin/Merged/Custom)/`AuditDimensionSpec` + `AuditScoreResult` 5-dimension /100 rubric (threshold 90). Generated at planning-draft confirm time (`audit_plan.rs`), persisted on `workflow.audit_plan`, executed by agent. Distinct from System A (`OrchestratorConfig::quality_gate_mode` off/shadow/warn/enforce).
- **Provider failover health** (`runtime.rs` + `config.rs`): `fallback_providers` build a resilient multi-provider LLM client; `get_provider_status`/`reset_provider` expose circuit-breaker state via `server/routes/provider_health.rs`.
- **Prompt state machine** (`types.rs`): `TerminalPromptStateMachine` debounces duplicate PTY prompts with a 5s same-prompt retry window and a 300s WaitingForApproval auto-reset (G07-008). Used by terminal `PromptWatcher`/`prompt_handler`.
