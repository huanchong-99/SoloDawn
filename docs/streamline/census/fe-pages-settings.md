## In-Flight Work Relevance

### G1: "Open in external IDE/editor" deletion candidate
`GeneralSettingsNew.tsx` lines 287-374 contain the full Editor settings card: EditorType selector, EditorAvailabilityIndicator, custom command input, and remote SSH host/user inputs. This is the ONLY settings UI for the IDE-open feature. If G1 cuts the editor-open feature, this entire `<SettingsCard>` block (including `useEditorAvailability` hook import, `EditorAvailabilityIndicator` import, `editorOptions` memo, `editorAvailability` state, and `supportsRemoteSsh` logic) should be removed from `GeneralSettingsNew.tsx`.

### G3: New QualitySettings page needed
No `QualitySettings` page exists yet in this scope. A new `QualitySettingsNew.tsx` would need to be created under `frontend/src/pages/ui-new/settings/`, exported from `ui-new/settings/index.ts`, and routed at `/settings/quality` in `App.tsx`. The `SettingsLayoutContainer.tsx` nav would also need a new nav item. Quality gate types/hooks exist at `frontend/src/hooks/useQualityGate.ts` and `frontend/src/components/quality/`.

### VS Code Webview Bridge
No webview bridge code detected in this scope. The editor-type setting in `GeneralSettingsNew` configures which editor to open files in, but the actual bridge/IPC is elsewhere.

### Quality Gate System (A)
`useQualityGate` hook exists and is in `hooks/index.ts` exports, used in `PipelineView.tsx`, `Board.tsx`, `Workflows.tsx`. No settings UI for quality gates in this scope yet — G3 gap confirmed.
