## Invisible Features

- **SharedTasks (VITE_ENABLE_SHARED_TASKS)**: `useProjectTasks` conditionally subscribes to ElectricSQL's `sharedTasksCollection` and the remote assignee API. Guarded by both `VITE_ENABLE_SHARED_TASKS === 'true'` and `remoteFeaturesEnabled`. Comment states the server-side APIs are disabled. The flag appears only in `useProjectTasks.ts` and a SonarCloud issues doc — no `.env` file sets it.
- **Feishu Sync in PlanningDraft**: `useTogglePlanningFeishuSync` in `usePlanningDraft.ts` exposes a Feishu channel integration for planning drafts, consumed by `FeishuChannelContainer`. Not exposed in main UI navigation.
- **Orchestrator Chat API**: `useSubmitOrchestratorChat` + `useOrchestratorMessages` in `useWorkflows.ts` expose a direct chat channel to the workflow orchestrator. Consumed by `ConciergeChatContainer`.
- **Planning Draft → Audit Doc upload**: `useUploadAuditDoc` / `useDeleteAuditDoc` are audit document management hooks in `usePlanningDraft.ts`; consumed by `AuditDocPanel`. Relevant to Quality Gate System B (AuditPlan).
- **Repo selection via disabled query**: `useAttemptRepo` stores `selectedRepoId` as a React Query cache entry with `enabled: false` and `staleTime: Infinity`. This is a hidden shared-state pattern not apparent from the hook name.
- **`usePrepareWorkflow` / `useMergeWorkflow`**: Lifecycle state transitions (`created→starting→ready` and `completed→merging`) with optimistic updates and rollback. Not re-exported from index.ts (accessible via direct import from `useWorkflows`).
