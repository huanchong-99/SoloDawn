## Invisible features

- **Redis message-bus backend** (`message_bus.rs::RedisBus`): full PubSub-backed bus for multi-container deployments, selected via `SOLODAWN_MESSAGE_BUS=redis` + `SOLODAWN_REDIS_URL`. Default is in-memory; Redis path only activates on env config. Functional but env-gated.
- **Crash recovery** (`persistence.rs`): orchestrator state is snapshotted to `workflow.orchestrator_state` and restored on `recover_workflow` (only for `running` workflows). Surfaced via `POST /recover`. Used.
- **Global terminal concurrency cap** (`constants.rs::configured_max_concurrent_terminals` + `SOLODAWN_MAX_CONCURRENT_TERMINALS`): conservative default 4 for Windows/WebView hosts; fails closed to default. Used.
- **Auto-appended quality-requirement prompts** (`constants.rs` suffixes): greenfield vs incremental quality standards silently appended to every terminal instruction (Quality Gate System A surface). Used.
- **Reserved `Checkpoint` completion status + `quality_pending` terminal status** (`types.rs`/`constants.rs`): drive the quality-gate checkpoint flow (System A/B). Used in agent.rs.
