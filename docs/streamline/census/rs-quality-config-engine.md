## Invisible features / reserved capability

- **SARIF auto-import** (`engine.rs::collect_sarif_issues`): scans `quality/sarif/`, `target/sarif/`, `.sarif/` for `*.sarif`/`*.json`, parses, maps tool name → `AnalyzerSource`, folds into the report as a `sarif-import` provider. Not surfaced in UI; runs every `run()`. Seems used (always invoked), but depends on tools dropping SARIF files into those dirs.
- **`QualityScope` merge-base/head reservation**: `IntroducedOnly{merge_base,head}` and `TaskBranch{merge_base,head}` and `RepoFinal{head}` are populated by agent.rs (real git merge-base computed) but the quality crate discards everything except `files`. This is the documented "provider-native scope is phase-2" compatibility shim — reserved, not dead.
- **Empty-scan fail-closed sentinel** (`MetricKey::QualityGateEmptyScan` + `quality_engine::empty_scan` rule id): meta-level safety net, never emitted by a provider; only synthesized in enforce mode when targets exist but no provider claims any gate metric.
- **`QualityGateMode::Off/Warn`**: `Off` → `run()` early-returns empty; `Warn` distinguished only via `is_enforcing()==false`. Mode plumbing lives in agent.rs (`self.config.quality_gate_mode`), not here.
