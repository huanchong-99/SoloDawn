# Streamline Ledger — Area: rust-backend

Synthesis of 30 unit-agent census reports (server routes, server core, orchestrator services, terminal services, git/host services, concierge, config, db models, db migrations, executors, utils, small crates, 3 gapfill passes).

Deduplicated, cross-linked, and ranked by removal priority. Removal priority is a function of `confidence × disposition × (low blastRadius)`. `delete`+`high`+isolated ranks highest; `keep` items are recorded for completeness but are not removal targets.

## Tooling incident note

**fast-context MCP (`mcp__fast-context__fast_context_search`) was UNAVAILABLE or quota-limited across nearly every unit.** Almost all units reported `resource_exhausted` (Windsurf backend quota, not auth) — many distinct trace IDs. Units that got 1 successful call before exhaustion: `rs-server-routes-config`, `rs-server-routes-planning-quality` (router-mount query only), `rs-server-core` (auth call sites), `rs-services-orch-llm-prompt` (both queries succeeded — the only fully-served unit), `rs-services-terminal-rest` (failed first call). **Consequence: virtually all cross-file caller/usage verification fell back to repo-wide Grep.** Grep-based "zero callers" claims may miss trait-object dispatch, macro-generated, or dynamic/string-dispatch call sites. Confidence levels below already reflect this caveat.
